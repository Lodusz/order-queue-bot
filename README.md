🎯 О проекте

Этот репозиторий содержит бэкенд-сервис, спроектированный с упором на чистую архитектуру. Здесь реализовано:

📱 Обработка команд через Telegram-бота (Long Polling)
🌐 Полноценный REST API для интеграции с фронтендом
🗄️ Надежное хранение данных в PostgreSQL с частичными индексами
🛑 Graceful Shutdown для безопасного завершения работы

Цель проекта: Показать навыки разработки масштабируемых микросервисов на Go.


📂 Структура репозитория

order-queue-bot/
┣ 📂 cmd/app/           # Точка входа, DI-контейнер, запуск горутин
┣ 📂 internal/domain/   # Ядро бизнес-логики и транспорт (HTTP/TG)
┣ 📂 migrations/        # SQL-скрипты для инициализации БД
┗ 📄 docker-compose.yml # Конфигурация для запуска инфраструктуры

⚙️ Требования для работы
Для локального запуска
Для корректной работы сервиса и базы данных необходимо установить Docker:
🐳 Скачайте Docker Desktop с официального сайта
📂 Откройте терминал в корневой папке репозитория
⚙️ Выполните команду docker-compose up -d для старта базы данных

Для работы приложения
Все основные зависимости подтянутся автоматически через go mod, но вам понадобятся:
Установленный Go версии 1.20+
Токен Telegram-бота (получается через @BotFather)
DBeaver или любой другой клиент для работы с БД (для наката миграций)


Как использовать проект
🧠 Работа с сервисом
Процесс локального тестирования содержит следующие шаги:
Клонирование репозитория на ваш компьютер
Запуск базы данных в Docker-контейнере
Применение SQL-скрипта к созданной базе orders_db
Запуск проекта командой go run cmd/app/main.go с указанием токена
